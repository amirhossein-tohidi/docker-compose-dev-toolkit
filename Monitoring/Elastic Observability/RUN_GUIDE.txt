Elastic Observability Run Guide

This setup includes Elasticsearch, Kibana, and Elastic APM Server.
It is for local development only, not production.

Kibana is the dashboard/UI for this stack.

1. Start Docker Desktop.

2. Open PowerShell or CMD in the cloned repository root, then enter this service folder:

cd ".\Monitoring\Elastic Observability"

3. Start Elasticsearch, Kibana, and APM Server:

docker compose up -d
docker compose ps

If your Docker installation uses the legacy Compose command, use:

docker-compose up -d
docker-compose ps

Default settings are already included in `.env`, so a fresh clone can run without creating extra files.

Optional:
If ports 9200, 5601, or 8201 are already in use, edit these values in `.env` before startup:

ELASTICSEARCH_PORT=9200
KIBANA_PORT=5601
APM_SERVER_PORT=8201

4. Open the tools:

Elasticsearch:
http://localhost:9200, or the value of `ELASTICSEARCH_PORT` if you changed it

Kibana:
http://localhost:5601, or the value of `KIBANA_PORT` if you changed it

Kibana Elasticsearch page:
http://localhost:5601/app/elasticsearch/getting_started

Kibana APM page:
http://localhost:5601/app/apm

In the Kibana menu, APM may appear under:

Observability -> APM and User Experience

Elastic APM Server:
http://localhost:8201, or the value of `APM_SERVER_PORT` if you changed it

5. Application APM endpoint:

Use this endpoint in applications running on the host machine:

http://localhost:8201

Use this endpoint in applications running inside the same Compose network:

http://apm-server:8200

This local setup does not require an APM secret token.

6. Quick tests:

Elasticsearch:

Open this URL in a browser:

http://localhost:9200/_cluster/health

APM Server:

Open this URL in a browser:

http://localhost:8201/

7. Stop without deleting data:

docker compose down

Or, with legacy Compose:

docker-compose down

Elastic data is stored in Docker named volumes:

elastic-observability-elasticsearch-data
elastic-observability-kibana-data

These volumes are created automatically by Docker. You do not need to create any data folder manually.

Restarting Docker Desktop, restarting Windows, or running `docker compose down` will not delete the data.

8. Fully reset Elastic Observability and delete its data:

docker compose down -v --remove-orphans

Or, with legacy Compose:

docker-compose down -v --remove-orphans

Important:
Elasticsearch security is disabled in this local stack to keep Kibana and APM simple for development. Do not use this configuration for production.
