Grafana Run Guide

Grafana is a dashboard and visualization tool.
It can connect to Prometheus, Elasticsearch, and many other datasources.

This setup provisions Prometheus as the default datasource.

1. Start Docker Desktop.

2. Open PowerShell or CMD in the cloned repository root, then enter this service folder:

cd ".\Monitoring\Grafana"

3. Start Grafana:

docker compose up -d
docker compose ps

If your Docker installation uses the legacy Compose command, use:

docker-compose up -d
docker-compose ps

Default settings are already included in `.env`, so a fresh clone can run without creating extra files.

Optional:
If port 3000 is already in use, edit `GRAFANA_PORT` in `.env`.

4. Open Grafana:

http://localhost:3000, or the value of `GRAFANA_PORT` if you changed it

5. Login:

Username:
admin

Password:
Abcd@1234

6. Prometheus datasource:

By default, Grafana connects to Prometheus at:

http://host.docker.internal:9090

This works well on Docker Desktop when Prometheus is running on the host at port 9090.

If you changed the Prometheus port, edit this value in `.env`:

GRAFANA_PROMETHEUS_URL=http://host.docker.internal:9090

7. Stop without deleting data:

docker compose down

Or, with legacy Compose:

docker-compose down

Grafana data is stored in a Docker named volume:

grafana-data

This volume is created automatically by Docker. You do not need to create any data folder manually.

Restarting Docker Desktop, restarting Windows, or running `docker compose down` will not delete the data.

8. Fully reset Grafana and delete its data:

docker compose down -v --remove-orphans

Or, with legacy Compose:

docker-compose down -v --remove-orphans
