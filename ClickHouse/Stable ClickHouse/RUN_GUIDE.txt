ClickHouse Run Guide

1. Start Docker Desktop.

2. Open PowerShell or CMD in the cloned repository root, then enter this service folder:

cd ".\ClickHouse\Stable ClickHouse"

3. Start ClickHouse:

docker compose up -d
docker compose ps

If your Docker installation uses the legacy Compose command, use:

docker-compose up -d
docker-compose ps

Default settings are already included in `.env`, so a fresh clone can run without creating extra files.

Optional:
If ports 8123 or 9000 are already in use, edit `CLICKHOUSE_HTTP_PORT` or `CLICKHOUSE_TCP_PORT` in `.env`.

4. Connection details:

HTTP:
http://localhost:8123, or the value of `CLICKHOUSE_HTTP_PORT` if you changed it

Play UI:
http://localhost:8123/play

Dashboard:
http://localhost:8123/dashboard

Health check:
http://localhost:8123/ping

Native TCP:
localhost:9000, or the value of `CLICKHOUSE_TCP_PORT` if you changed it

Database:
default

Username:
admin

Password:
Abcd@1234

5. Quick test from inside the container:

docker compose exec -T clickhouse clickhouse-client --user admin --password "Abcd@1234" --query "SELECT version(), now()"

6. Open the interactive ClickHouse client:

docker compose exec clickhouse clickhouse-client --user admin --password "Abcd@1234"

7. Stop without deleting data:

docker compose down

Or, with legacy Compose:

docker-compose down

ClickHouse data is stored in Docker named volumes:

clickhouse-data
clickhouse-logs

These volumes are created automatically by Docker. You do not need to create any data folder manually.

Restarting Docker Desktop, restarting Windows, or running `docker compose down` will not delete the data.

8. Fully reset ClickHouse and delete its data:

docker compose down -v --remove-orphans

Or, with legacy Compose:

docker-compose down -v --remove-orphans
