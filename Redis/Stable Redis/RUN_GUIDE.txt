Redis Run Guide

1. Start Docker Desktop.

2. Open PowerShell or CMD in the cloned repository root, then enter this service folder:

cd ".\Redis\Stable Redis"

3. Start Redis:

docker compose up -d
docker compose ps

If your Docker installation uses the legacy Compose command, use:

docker-compose up -d
docker-compose ps

Default settings are already included in `.env`, so a fresh clone can run without creating extra files.

Optional:
If port 6379 is already in use, edit `REDIS_PORT` in `.env`.

4. Connection details:

Host:
localhost

Port:
6379, or the value of `REDIS_PORT` if you changed it

Password:
Abcd@1234

Connection String:
redis://:Abcd%401234@localhost:6379/0

5. Quick test:

docker compose exec -T redis redis-cli -a "Abcd@1234" ping

6. Open redis-cli:

docker compose exec redis redis-cli -a "Abcd@1234"

7. Stop without deleting data:

docker compose down

Or, with legacy Compose:

docker-compose down

Redis data is stored in a Docker named volume:

redis-data

This volume is created automatically by Docker. You do not need to create any data folder manually.

Restarting Docker Desktop, restarting Windows, or running `docker compose down` will not delete the data.

8. Fully reset Redis and delete its data:

docker compose down -v --remove-orphans

Or, with legacy Compose:

docker-compose down -v --remove-orphans
