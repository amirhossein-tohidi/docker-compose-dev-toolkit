SQL Server Run Guide

1. Start Docker Desktop.

2. Open PowerShell or CMD in the cloned repository root, then enter this service folder:

cd ".\Sql\Stable SQL Server"

3. Start SQL Server:

docker compose up -d
docker compose ps

If your Docker installation uses the legacy Compose command, use:

docker-compose up -d
docker-compose ps

Default settings are already included in `.env`, so a fresh clone can run without creating extra files.

Optional:
If port 1433 is already in use, edit `SQLSERVER_PORT` in `.env`.

4. Connection details:

Server:
localhost,1433, or the value of `SQLSERVER_PORT` if you changed it

Username:
sa

Password:
Abcd@1234

Trust Server Certificate:
True

5. Quick test:

docker compose exec -T sqlserver /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P "Abcd@1234" -C -Q "SELECT @@VERSION"

If the `mssql-tools18` path does not exist, use:

docker compose exec -T sqlserver /opt/mssql-tools/bin/sqlcmd -S localhost -U sa -P "Abcd@1234" -Q "SELECT @@VERSION"

6. Open sqlcmd:

docker compose exec sqlserver /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P "Abcd@1234" -C

7. Stop without deleting data:

docker compose down

Or, with legacy Compose:

docker-compose down

SQL Server data is stored in a Docker named volume:

sqlserver-data

This volume is created automatically by Docker. You do not need to create any data folder manually.

Restarting Docker Desktop, restarting Windows, or running `docker compose down` will not delete the data.

8. Fully reset SQL Server and delete its data:

docker compose down -v --remove-orphans

Or, with legacy Compose:

docker-compose down -v --remove-orphans
